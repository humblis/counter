 -앱 설명:
지폐 금액를 빠르게 입력하고 각 지폐의 수량과 합계를 계산 하는 것을 도와주는 입력 및 조회 도구임.

- 앱의 타깃 디바이스는 스마트폰 등 모바일 스크린임.
물론 웹 브라우저를 사용하는 모든 장치에서 사용이 가능하도록 함.

- 화면을 구성은 다음과 같음.
1. 키패드 부분:
금액을 입력하기 위한 숫자 키패드 0~9,
입력된 모든 금액을 지우는 C 키,
입력된 금액의 마지막 하나를 지우는 백스페이스 키,
기본 숫자 키패드와 C, 백스페이스 키는 3x4 형식으로 구현함.
자주 입력되는 금액을 지정한 사용자 지정키 12개 (예: 12, 16, 18, 20, 22, 26, 28, 32, 36, 38, 60, 60)가 있음.
자주 입력되는 금액을 지정한 키를 '사용자 지정키'라고 부르며 사용자 지정키의 배경색은 기본 숫자 키패드와 다르게 구별함.
사용자 지정키를 누를 경우, 반드시 기존에 입력된 숫자 전부를 지우고 새로 입력함.
// 숫자 키패드의 숫자가 10 미만일 경우에만 추가로 입력하고 10 이상일 경우에는 기존의 내용을 지우고 누른 숫자의 값만 입력함.

2. 기능 버튼:
인덱스를 임의로 지정하기 위한 인덱스 버튼,
금액을 표시하고 입력하기 위한 금액 버튼,
저장된 데이터를 액셀 파일로 다운로드 하기위한 다운로드 버튼,
개발용 버전에서만 사용되는 캐시 삭제 버튼이 있음.

3. 정보를 표시하는 부분:
저장된 금액의 합계을 표시하는 부분이 있음.
이 부분에는 표에 나타난 금액의 합계를 표시함.

4. 저장된 데이터를 표시하는 부분:
로컬 스토리지에 저장된 금액을 보여주는 테이블이 있음.
테이블의 기본 정렬은 idx를 기준으로 내림차순으로 함.
테이블의 헤드는 idx, amount, name, time, actions가 있음.
해드 중 idx, amount, name, time에는 해당 컬럼을 누르면 오름차순과 내림차순으로 정렬되는 기능을 토글 방식으로 구현함.
actions 컬럼에서 편집 버튼을 누르면 해당 행의 데이터를 편집할 수 있도록 하고 편집을 마치고 나서 완료 버튼을 누르면 저장되도록 함.
그 옆에 해당 행을 삭제할 수 있는 삭제 버튼을 넣음. 단, 삭제 버튼은 기능을 수행하기 전에 사용자에게 확인하는 과정을 넣음.
페이지가 로딩되면 자동으로 저장된 데이터가 있는지 확인하여 저장된 값들을 테이블 형식으로 보여줌.

- 코드 구현은 다음과 같음.
웹 앱의 기본 페이지 (index.html), 자바스크립트 (scripts/ceremoney.js), 스타일 (styles/ceremoney.css), 
저장된 데이터 테이블의 리스트를 관리하는 페이지 (data.html)
테이블이 여러 개일 수 있으므로 테이블을 관리하는 테이블도 생성함.
기본 페이지의 좌측 상단에 데이터 관리 페이지로 이동할 수 있는 메뉴가 있음.
데이터 관리 페이지의 좌측 상단에 기본 페이지로 이동할 수 있는 메뉴가 있음.
금액 버튼을 누르면 다음의 컬럼을 가지는 데이터를 브라우저의 로컬 스토리지의 데이블에 저장함.
1. 금액 인덱스 번호 (moneyId, 1부터 시작함)
2. 이름 (name, 입력시에는 알 수 없으나 나중에 편집해서 입력이 가능함)
3. 버튼 내용에 있는 금액 (amount)
4. 데이터가 최초 입력된 생성 일시 (created datetime)
5. 데이터가 입력된 일시 (created datetime)
6. 데이터가 수정된 일시 (updated datetime)
7. 이벤트 아이디 (eventId, 해당 금액을 받은 행사)
7. 설명 (note)

표를 조회할 경우 현재 테이블의 모든 데이터의 금액 인덱스, 금액, 데이터가 최초 입력된 일시 (짧은 형식, YY-MM-DD HH:MM:SS), 수정한 일시 컬럼에 대해 표 형식으로 표시함. 일시를 나타내는 두 컬럼을 하나의 셀로 만들고 셀 내부에 위 아래로 배치하여 공간을 절약함.
편집 상태가 아닐 경우, 즉, 편집 버튼을 누르기 전에는 셀 내용을 조작할 수 없도록 함.

표의 헤드에는 컬럼 이름이 짧게 나타나 있고 그 컬럼들은 클릭이 가능하도록 하고 클릭할 때마다 토글 방식으로 오름차순, 내림차순이 번갈아 가면서 표를 정렬함.

편집 버튼을 누르는 경우 금액 인덱스 (moneyId)는 변경할 수 없고 금액과 이름을 편집한 후 저장 버튼을 누르면 저장되도록 함.
편집 버튼을 누르는 경우 해당 셀을 편집 가능하도록 만들고, 수정 일시 값은 저장 버튼을 눌렀을 때의 시간으로 함.

다운로드 버튼을 누르는 경우, 로컬 스토리지에 저장된 데이터를 액셀 파일로 다운로드함.
